## 🎮 모드 소개

| 모드 | 설명 | 인원 | 특징 |
|------|------|------|------|
| **AI 모드** 🧠 | 1인용 캠페인 | 플레이어 1 + AI 1~9명 | 레벨 1→9 도전, 사망 시 현재 레벨 재도전 |
| **인간 멀티** 👥 | 공유 룸 PvP | 인간 최대 9 + 봇 충원(총 9) | 10초 카운트다운 후 시작, 룸 자동 생성 |

**AI 모드**는 순차적 난이도 상승: 레벨 1(AI 1명) → 레벨 9(AI 9명). 승리 시 다음 레벨로 진급하며, 사망 시 현재 레벨을 재도전합니다. 도달한 최고 레벨이 오늘의 AI 랭킹에 기록됩니다.

**인간 멀티 모드**는 자동 생성되는 룸에서 최대 9명이 경쟁합니다. 인원이 부족하면 봇이 자리를 채우고, 10초 카운트다운 후 자동 시작됩니다. 9명이 모두 모이면 즉시 시작합니다.

## 🚀 빠른 시작

### 📱 모바일 (30초)

```bash
1. PC에서 npm run launch  # 서버 실행
2. 폰 브라우저 → http://<PC_IP>:3000
3. 이름 입력 → AI 모드 선택
4. 화면 터치 유지 → 자동 전투 시작!
```

> **한손 모드**가 기본입니다. 손을 떼면 정지, 다시 터치하면 재개됩니다.

### 🖥️ 데스크탑 (30초)

```bash
1. npm run launch
2. http://localhost:3000 접속
3. 이름 입력 → 모드 선택
4. WASD 이동 + 마우스 클릭 사격!
```

## 🏃 서버 실행 가이드

### 원 클릭 실행 (권장)

```bash
npm run launch     # = node launch.js
```

의존성 확인 → 서버 기동 → 헬스체크 → 브라우저 자동 오픈까지 한 번에 처리합니다.

| 플랫폼 | 실행 방법 |
|--------|----------|
| **Windows** | `start.bat` 더블클릭 또는 `npm run launch` |
| **macOS/Linux** | `./start.sh` 또는 `npm run launch` |

**옵션:**
- `node launch.js --no-open` — 브라우저 자동 오픈 생략
- `PORT=4000 npm run launch` — 포트 지정
- 종료: `Ctrl+C`

### 수동 실행

```bash
npm install        # 최초 1회 (express, socket.io, helmet, compression)
npm start          # = node server.js  → http://localhost:3000
```

### 환경 변수

| 변수 | 기본값 | 설명 |
|------|--------|------|
| `PORT` | `3000` | HTTP 수신 포트 |
| `NODE_ENV` | — | `production` 시 정적 자산 캐시(maxAge=1d) 활성화 |
| `CORS_ORIGIN` | 동일출처만 | Socket.io 허용 출처 (`*` 또는 콤마 목록) |
| `MAX_CONN_PER_IP` | `8` | IP당 동시 연결 상한 (DoS 완화) |
| `MAX_ROOMS` | `200` | 전체 룸 수 상한 (자원 고갈 방지) |

### 모바일에서 테스트 (같은 와이파이)

1. PC에서 `npm start`
2. 폰 브라우저 → `http://<PC_IP>:3000` (예: `http://192.168.0.18:3000`)
3. IP 확인: Windows `ipconfig`, macOS `ifconfig | grep inet`
4. Windows 방화벽에서 Node.js의 3000번 포트(개인 네트워크) 허용 필요 시

### 멀티플레이 확인

- **같은 PC 2탭**: 브라우저 탭 2개를 `http://localhost:3000`으로 열고 둘 다 **인간 멀티** 선택 → 같은 룸에 배정되어 전투 가능
- **라운드 흐름**: `대기(로비) → 플레이 → 최후 1인 → 결과 → 다음 라운드`
- **도중 접속/사망**: 관전 모드 → 다음 라운드 자동 참여
- **로비 시간**: 멀티 10초 / AI 3초

## 🎮 조작법

### 📱 모바일 - 한손 모드 (기본, 🔥 추천)

> **터치 한 번으로 완전 자동 플레이!** 초보자부터 고수까지 누구나 쉽게 즐길 수 있습니다.

| 입력 | 동작 |
|------|------|
| 👆 **화면 터치 유지** | 자동 모드 활성화 (AI가 모든 조작 처리) |
| ✋ **터치 해제** | 자동 모드 중지 |
| **− / +** | 줌 아웃 / 인 |
| **⚙️** | 설정 패널 열기 |
| **💣** | 수류탄 투척 (보유 시) |
| 👆👆 두 손가락 핀치 | 줌 인 / 아웃 |

**자동 모드가 처리하는 것:**

| 기능 | 설명 |
|------|------|
| 🎯 **자동 조준** | 위협도 기반 최적 타겟 선정 (저체력·접근 적 우선) |
| 🚶 **자동 이동** | 존 회피(최우선) + 타겟 추적/후퇴 + 스트레이프 회피 |
| 🔄 **자동 무기 전환** | 거리별 최적 무기 자동 선택 (근접→샷건, 중거리→SMG, 원거리→권총) |
| 🔫 **자동 사격** | 무기별 유효 사거리 내에서 자동 발사 |
| 💣 **자동 수류탄** | 최적 거리에서 쿨다운 기반 자동 투척 |
| 🧱 **장애물 회피** | 장애물 감지 시 자동 우회 (2025년 v2.2 추가) |
| ⚡ **예측 조준** | 탄속과 타겟 속도를 고려한 리드 사격 |

**화면 하단 인디케이터:**
- `AUTO` (🟡 황금색 펄스) — 자동 모드 활성
- `TOUCH` (⚪ 회색) — 대기 상태

**💡 팁:**
- 공격성 70%+ → 적극적 추격 / 30%- → 신중 플레이 (생존 우선)
- 존이 좁아질수록 공격성을 낮추는 것이 유리합니다
- 공격성은 설정(⚙️)에서 0~100% 슬라이더로 조절 가능

## 🆕 주요 기능

### 한손 모드 완전 자동화 (v2.2)

`client/touch.js`의 `_syncOnehand()`가 실시간으로 게임플레이를 자동 처리합니다:

```
30Hz 루프: 타겟 선정 → 예측 조준 → 존/장애물 회피 이동 → 무기 전환 → 사격 → 수류탄
```

> 서버는 `{moveX, moveY, angle, firing, reload, sprint}` 입력만 수신하므로 **서버 권위적 보안을 유지**합니다.

#### 개선 사항 (v2.2 ~ v2.3)
| 개선 | 설명 |
|------|------|
| ✅ **터치 응답성 개선** | 터치 즉시 이동 시작 (기존 1~2프레임 지연 해결) |
| ✅ **정확한 무기 전환** | `Net.sendSwitchWeapon(key)` 직접 호출로 원하는 무기로 즉시 전환 |
| ✅ **장애물 회피** | 장애물 감지 시 자동 우회 (선호→스트레이프→원래 방향) + 방향 캐싱으로 지터 제거 |
| ✅ **정확한 존 중심** | 서버의 실제 존 중심 좌표 사용 (고정 중심→동적 중심) |
| ✅ **부드러운 움직임** | 지수 평활화 이동(0.35) + 프레임 카운터 기반 스트레이프로 마이크로지터 제거 |
| ✅ **안정적인 타겟팅** | 타겟 락으로 빠른 전환 방지 + 조준 방향 평활화로 에임 떨림 제거 |
| ✅ **존 회피 안정화** | 히스테리시스 임계값(120px/180px)으로 플립플랍 방지 |
| ✅ **멀티페이지 튜토리얼** | 시작 화면에 이전/다음/건너뛰기 + 페이지 도트, 모바일/데스크탑 분기 콘텐츠 |
| ✅ **맞춤형 HUD 힌트** | 게임 첫 10초간 조작 체계별 컨텍스트 힌트 표시 |
| ✅ **첫 방문 자동 튜토리얼** | `localStorage` 플래그 기반 첫 방문 시 600ms 후 자동 표시 |

### 핵심 설계 원칙

**서버 권위적 (Server Authoritative)**

모든 시뮬레이션은 서버에서 실행됩니다. 클라이언트는 입력을 30Hz로 송신하고, 서버의 스냅샷(20Hz)을 수신하여 렌더링만 수행합니다. 입력은 `{moveX, moveY, angle, firing, reload, sprint}` 계약으로, 서버에서 `sanitizeController()`로 정규화됩니다.

**입력 처리 우선순위:**

```
게임패드(active) > 터치(active 또는 HYBRID_MS 500ms 이내) > 키보드/마우스
```

**네트워크 모델**

| 구성 요소 | 방식 |
|-----------|------|
| 입력 송신 | 30Hz (`setInterval`, 로컬 예측용 rAF 샘플링) |
| 스냅샷 수신 | ~20Hz (서버 부하 기반) |
| 엔티티 보간 | 지수 스무딩 (k = 1 - e^(-15*dt), 20Hz 지터 숨김) |
| 자기 예측 | 로컬 입력 기반 위치 예측 + 장애물 충돌 포함 |
| 보정 | PREDICT_RECONCILE=110px 이상 즉시 스냅, SOFT=28px 이상 블렌드 |
| 재접속 유예 | 20초, 같은 clientId/mode로 복구 |

**룸/모드 시스템**

```
각 룸 → 독립 GameSim 인스턴스
├── AI 룸: 인간 1 + 봇 level개 (레벨 1~9 캠페인)
└── Multi 룸: 인간 최대 9 + 봇 충원 (총 9, 10초 로비)

상태머신: lobby → playing → roundover → (recycle)
```

**클라이언트 예측 상세**

- 자기 엔티티는 보간 루프에서 **제외** — 보간이 예측 위치를 서버 지연 위치로 당기는 mushy 현상 방지
- 자기 위치는 로컬 입력으로 매 프레임 예측 유지
- 서버 스냅샷과 110px 이상 차이 → 즉시 스냅 (리스폰/순간이동)
- 28px ~ 110px → 부드러운 블렌딩 (고무줄 효과 완화)

## 🛡️ 보안 / 운영

| 항목 | 적용 내용 |
|------|----------|
| **정적 파일 화이트리스트** | `/client`, `/css`, `/shared`, `index.html`만 서빙. 나머지는 404 |
| **보안 헤더** | helmet (CSP는 캔버스 호환성 위해 비활성), compression, trust proxy |
| **CORS** | `CORS_ORIGIN` 환경변수로 제한 (미설정=동일출처) |
| **속도 제한** | MAX_CONN_PER_IP, MAX_ROOMS, 입력 30Hz 초과 드랍, Socket.io maxHttpBufferSize |
| **입력 검증** | `sanitizeController()` — NaN/비정상 입력 차단 (기초 안티치트) |
| **메모리 관리** | disconnect 유예 타이머 만료 시 핸들 해제 (GC 허용) |
| **그레이스풀 셧다운** | SIGTERM/SIGINT → io.close() + httpServer.close() |
| **일간 리셋** | UTC 자정 기준 리더보드 초기화 |

> ⚠️ 현재는 "최소 방어" 단계입니다. `clientId`는 서명/토큰 없는 난수이며, 광고 보상은 서버 횟수 제한만 있습니다. 대규모 운영 전 토큰 인증 및 광고 SSV(Server-Side Verification) 도입이 필요합니다.

## 📊 밸런스 및 개발

### 레벨 밸런스 측정

헤드리스 시뮬레이터로 AI 레벨별 승률/피해/생존시간/킬을 측정합니다:

```bash
TRIALS=40 AIM=0.07 npm run balance
# TRIALS=시뮬레이션 수, AIM=플레이어 숙련도(aimError)
```

중간 실력(aimError≈0.07) 기준:
```
L1 ≈ 88% → L5 ≈ 50% → L9 ≈ 30~40% (단조적 감소)
```

튜닝 지점:
- `shared/player.js` → `Bot.setSkill()` (레벨별 능력치 매핑)
- `shared/sim.js` → `startRound()` (난이도 t 산출 · 시작 무기)

### 봇 난이도 스케일링

AI 모드 레벨 상승에 따라 봇 능력치가 선형 증가합니다:

| 능력 | 레벨 1 | 레벨 5 | 레벨 9 |
|------|:-----:|:-----:|:-----:|
| 속도 | 낮음 | 중간 | 높음 |
| 조준 정확도 | 낮음 | 중간 | 높음 |
| 표적 리드 | 없음 | 중간 | 높음 |
| 사격 의지 | 보수적 | 보통 | 적극적 |
| 체력 | 기본 | + | ++ |
| 시작 무기 | 권총 | SMG | 샷건 |

> Multi 봇은 난이도 중간(t≈0.45) 고정.

### 서버 고정 타임스텝

```
accumulator 방식: 벽시계 delta 누적 → 1/60 초 단위로 다중 스텝 실행
→ 이벤트루프 지연에도 실시간 동기 유지, 스파이크는 clamp
```

### UI 커스터마이징 포인트

- **엔티티**: 입체 하이라이트 + 무기 금속광 + 아군 흰색 글로우 + 둥근 글로스 체력바
- **존**: 펄스 글로우 테두리
- **픽업**: 글로우 + 부양 + 타입별 색/아이콘
- **총알**: 잔광(outer) + 밝은 코어(inner)
- **HUD**: 체력바(그라데이션·눈금·저체력 펄스), 무기 슬롯(컬러 액센트), 미니맵, 킬 피드 패널
- **조준선**: 무기 산탄도 반영 커스텀 조준선 (데스크탑)
- **오버레이**: 로비/결과 카드 패널 + AI 레벨 진행 바(1~9)

## ❓ 문제 해결

### 서버 실행 문제

| 문제 | 해결 방법 |
|------|----------|
| `EADDRINUSE` 에러 | 이미 실행 중인 서버 종료 후 재시작, 또는 `PORT=4000 npm start` |
| `npm install` 실패 | Node.js 18+ 확인. `node --version` |
| 브라우저 자동 오픈 안 됨 | `node launch.js --no-open` 후 수동 접속 |

### 접속 문제

| 문제 | 해결 방법 |
|------|----------|
| 모바일에서 접속 안 됨 | 같은 와이파이 확인. 방화벽에서 3000번 포트 허용 |
| "Server full" 메시지 | `MAX_ROOMS` 환경변수 증가 (기본 200) |
| 연결 끊김 반복 | 네트워크 안정성 확인. 서버 로그에서 에러 확인 |
| 핑이 너무 높음 | `server.js`의 `PING_INTERVAL` 확인 (기본 2초) |

### 게임플레이 문제

| 문제 | 해결 방법 |
|------|----------|
| 한손 모드가 반응하지 않음 | 화면 하단 `TOUCH` 인디케이터 확인. 터치 지원 기기 필요 |
| 캐릭터가 움직이지 않음 | 조작 체계 확인 (설정 → 한손/캐주얼/듀얼). AI 모드에서만 전투 활성화 |
| 무기가 전환되지 않음 | 무기 픽업 필요 (시작 시 권총만 소지). 보유 무기만 전환 가능 |
| 수류탄이 안 던져짐 | 픽업으로 보급 필요. 최대 3개 |
| 화면이 너무 작게/크게 보임 | −/+ 버튼 또는 마우스 휠로 줌 조절 |
| 소리가 안 들림 | 🔊 버튼으로 음소거 해제. 브라우저에서 오디오 권한 허용 |
