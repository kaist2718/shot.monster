# Surviv 스타일 2D 배틀로얄 — v2 실시간 멀티플레이

Node.js + Socket.io 기반 **서버 권위적 실시간 멀티플레이** 배틀로얄. 시작 시 **모드 선택**(AI / 인간 멀티). 각 룸은 독립 `GameSim` 인스턴스.

- **AI 모드** — 1인용 캠페인. 레벨 1(=AI 1명) ~ 레벨 9(=AI 9명). 승리 시 다음 레벨, **사망 시 현재 레벨 재도전**. 도달한 최고 레벨을 오늘의 랭킹에 기록.
- **인간 멀티 모드** — 공유 룸. **방당 인간 9명 제한**, 부족 인원은 봇이 채워 총 9명 유지. 여러 룸이 자동 생성되며 브라우저에 룸별 인원수 표시. 인원이 모자라면 **10초 카운트다운 후 자동 시작**(9명 도달 시 즉시 시작).

## 실행

### 원 클릭(권장) — 서버 실행 + 브라우저 자동 오픈

의존성 확인 → 서버 기동 → 헬스체크 → 기본 브라우저 자동 열기를 한 번에:

```bash
npm run launch     # = node launch.js   (PORT 환경변수로 포트 변경 가능)
```

- **Windows**: `start.bat` 더블클릭 (또는 `npm run launch`)
- **macOS/Linux**: `./start.sh` (또는 `npm run launch`)
- 옵션: `node launch.js --no-open` (브라우저 자동 오픈 생략), `PORT=4000 npm run launch` (포트 지정)
- 종료: 콘솔에서 `Ctrl-C`

### 수동 실행(종전 방식)

```bash
npm install        # express, socket.io, helmet, compression 설치 (최초 1회)
npm start          # = node server.js  -> http://localhost:3000
```

브라우저에서 `http://localhost:3000` 접속. 최초에 **이름/국가 입력 시작 화면** → **모드 선택(AI / 인간 멀티)**. (기본 영문, 우측 상단 **EN/한국어 토글**.) 선택값은 `localStorage`에 저장되어 다음 접속부터 "계속" 한 번으로 입장.

### 모바일 설치형 앱(PWA)

홈화면에 추가해 **전체화면 스탠드얼론 앱**처럼 실행 가능(오프라인 셸 지원):

- **Android(Chrome)**: 메뉴 → "앱 설치" / "홈 화면에 추가". 매니페스트 + 192/512 아이콘 + 서비스워커 갖춤.
- **iOS(Safari)**: 공유 → "홈 화면에 추가". `apple-mobile-web-app-capable` + apple-touch-icon 적용.
- 설치 후 상단 주소창/브라우저 UI 없이 게임이 실행되고, 끊긴 네트워크에서도 앱 셸이 로드됩니다(실시간 데이터는 여전히 서버 필요).
- 앱 아이콘 재생성(수정 시): `npm run icons` (`scripts/gen-icons.cjs` — 의존성 없는 순수 Node PNG 인코더).

### 환경 변수

| 변수 | 기본 | 설명 |
|---|---|---|
| `PORT` | `3000` | 수신 포트 |
| `NODE_ENV` | — | `production` 시 정적 자산 캐시(`maxAge=1d`) 활성화 |
| `CORS_ORIGIN` | (미설정=동일출처만) | Socket.io 허용 출처. `*` 또는 `https://a.com,https://b.com` 콤마 목록 |
| `MAX_CONN_PER_IP` | `8` | IP당 동시 연결 수 상한(DoS 완화) |
| `MAX_ROOMS` | `200` | 전체 룸 수 상한(자원 고갈 방지) |

### 폰에서 테스트 (같은 와이파이)
1. PC에서 `npm start`
2. 폰 브라우저 → `http://<PC_IP>:3000` (예: `http://192.168.0.18:3000`)
3. 안 되면 Windows 방화벽에서 Node의 3000번 포트(개인 네트워크) 허용

### 멀티플레이 확인
- **같은 PC**: 브라우저 탭 2개를 `http://localhost:3000` 로 열고 둘 다 **인간 멀티** 선택 → 같은 룸에 배정되어 같은 라운드에서 싸움(룸 브라우저에서 인원수 확인).
- 라운드 흐름: `대기(로비) → 플레이 → 마지막 1인 → 결과 → 다음 라운드`. (멀티 로비 10초, AI 로비 3초.)
- 도중 접속 / 라운드 중 사망 시 **관전** → 다음 라운드 자동 참여.

## 아키텍처

```
shared/   서버+클라이언트 공용 ESM (DOM 무의존)
  config.js  utils.js  entity.js  world.js  player.js  sim.js  countries.js
client/   브라우저 전용 ESM
  input.js  touch.js  camera.js  render.js  net.js  ui.js  dom.js
  particles.js  sound.js  ad.js  start.js  mode.js  i18n.js  main.js
  mobile.js  boot.js  gamepad.js  aimassist.js
server.js Express(정적 화이트리스트) + Socket.io + 다중 룸 + 60Hz 시뮬/20Hz 스냅샷
launch.js 원 클릭 실행기(의존성 확인→기동→헬스체크→브라우저 오픈)
public/   PWA 자산(manifest.webmanifest · sw.js · icons/)
scripts/  gen-icons.cjs(의존성 없는 PNG 아이콘 생성기)
start.bat / start.sh  더블클릭/터미널 원 클릭 실행
```

- **서버 권위적**: 시뮬레이션(`sim.js`의 `GameSim`)은 서버에서만. 클라는 입력 송신(30Hz) + 스냅샷 수신 렌더링만.
- **컨트롤러 계약** `{moveX,moveY,angle,firing,reload,sprint}`: 클라가 빌드해 송신, 서버가 적용. **서버에서 `sanitizeController`로 정규화**(NaN/비정상 입력 차단 — 기초 안티치트). 반자동 사격은 서버 엣지 검출.
- **총알 동기화**: 스냅샷에 속도 포함, 클라가 수신 시각 기반 외삽으로 부드럽게. 렌더는 점 대신 속도 역방향 **짧은 트레일**로 가시성 확보(`client/render.js`).
- **보간 + 클라 예측**: 엔티티 위치 지수 스무딩(20Hz 지터 숨김). **내 플레이어는 로컬 입력으로 위치 예측**(장애물 충돌 포함)하고 서버 스냅샷으로 주기 보정(크게 벌어지면 즉시 스냩). 리스폰 점프는 즉시 스냅. **자기 엔티티는 보간 루프에서 제외** — 보간이 예측 위치를 서버 지연 위치로 당겨 입력이 둔해지는(mushy) 현상 방지.
- **정적 데이터 분리**: 장애물은 라운드 시작 시 `roundStart` 이벤트로 한 번만 전송(스냅샷은 가볍게). 파괴된 크레이트는 `obstacleDestroyed`로 전달.
- **다중 룸 + 모드**: 각 룸은 독립 `GameSim`(`{ id, mode, sim, controllers }`). 소켓은 자기 룸의 socket.io 룸에 조인해 `io.to(roomId)`로 스냅샷/이벤트 송수신(글로벌 보드만 전역). **AI 룸**=인간 1 + 봇 level 개(레벨 1~9 캠페인, 사망 시 재도전), **multi 룸**=인간 최대 9 + 봇 충원(총 9), 10초 로비 후 자동 시작·자동 생성. 모드는 `auth.mode`로 전송(부재 시 multi).
- **엔티티 ID는 전역 고유**: 모든 룸의 player/bot id가 globally unique(전역 시퀀스) — 룸 간 `yourId` 충돌 방지.
- **매치 상태머신**: 룸별 `lobby → playing → roundover` 순환. 최소 인간 1명 있어야 라운드 시작(빈 봇 전쟁 방지). AI 룸은 인간이 유예 중(탭 닫힘)이면 시뮬 정지 — 복귀 시 이어서. **유예(grace) 중인 인간은 라운드 스폰에서 제외**(무방비 표적/봇 수 밀어내기 방지)하고 다음 라운드 복귀 시 스폰.
- **모드 인식 재접속**: 클라는 `localStorage`의 영속 `clientId`를 `auth`로 보내, 끊김 후 유예(20s) 내 **같은 모드** 룸으로 재접속 시 동일 플레이어(이름/점수/진행) 유지. **같은 모드 재전환(`joinAI`/빠른입장 `joinMulti`)은 no-op** — AI 캠페인 레벨/룸이 보존됨(중복 클릭으로 진행도가 날아가는 버그 방지).
- **2탭 = 별개 플레이어**: 같은 브라우저 2탭(같은 clientId)은 별개 player로 스폰. **오늘의 랭킹/코인 보드는 `player.id`(세션) 단위**로 분리되어 2탭이 같은 보드 엔트리에 매 틱 덮어쓰는 정합성 버그가 없다. (유예 내 재접속은 같은 player.id 복구라 점수 누적 유지.)
- **피드백**: 킬 피드(kill 이벤트), 히트마커(hit 이벤트), 핑(RTT) 표시, 총구 화염, 데미지 시 화면 흔들림(선형 감쇠). 적 타격 시 **뜨는 데미지 숫자**(hit 이벤트가 `damage` 휴대).
- **사운드**(`client/sound.js`): WebAudio 합성 SFX(자산 없이 생성). 발사·적중·킬·재장전·픽업·피격·사망 소리. 노이즈 층은 **캐시된 버퍼 재사용**(매 발사마다 버퍼 재할당하는 GC 부하 제거). 발사체는 화면 좌우 위치로 패닝 + 거리 따라 볼륨 감쇠. 좌상단 🔊/🔇 버튼으로 음소거(`localStorage` 저장), 최초 제스처에서 컨텍스트 생성.
- **파티클**(`client/particles.js`): 총구 불꽃·탄피, 탄 적중 불꽃, 피격 혈흔, 사망 파편, 픽업 반짝임, 뜨는 데미지 숫자(월드 좌표, 줌에 맞춰 축소, 폰트 스트링 캐싱).
- **무기 전환**: `1`(권총) / `2`(SMG) / `3`(샷건, 픽업 시) — 서버 검증. 모바일은 🔫 버튼이 보유 무기를 순환. HUD 슬롯에 보유 무기 표시.
- **플레이어 충돌**: 엔티티가 서로 통과하지 않음 — `stepPlay`에서 원-원 겹침 분리(2회 반복, 장애물 재검사로 벽 밀림 방지). 서버 권위적이므로 전 클라이언트 일관.
- **존(수축 원) 단계적 강화**: 반경이 좁아질수록 바깥 DPS가 `ZONE_DPS_START`(8) → `ZONE_DPS_END`(26)로 증가(`Zone.currentDps()`). 후반 긴장감.
- **봇 난이도 스케일링 + 존 회피(`player.js` Bot)**: AI 모드는 레벨이 오를수록 봇 능력치(속도/조준 정확도/표적 리드/사격 의지/체력/시작 무기)가 상승(`Bot.setSkill(t)`, `t=(level-1)/8`+소분산). multi 충원 봇은 중간(t≈0.45) 고정. 봇은 존 가장자리에서 전투보다 중심 이탈을 우선해 **존 사냥 치즈를 방지**.
- **서버 고정 타임스텝 누적기(accumulator)**: 벽시계 delta를 누적해 1/60 단위로 다중 스텝 → 이벤트루프 지연에도 실시간 동기 유지, 스파이크는 클램프.
- **화면 줌 조정(사용자)**: 모바일은 **상단 −/+ 버튼** 또는 **두 손가락 핀치**(스틱 손가락은 핀치에서 제외), 데스크탑은 **마우스 휠**(줌인/아웃 모두). 선택값은 **기기 클래스별**(터치/마우스) `localStorage`에 저장. `VIEW.MIN/MAX_ZOOM` 범위 내.
- **모바일 컨트롤(`client/touch.js`)**: 하단 우측에 **🔫 무기교체·R 재장전**(크게, 옵션으로 🔴 사격/🏃 달리기 토글 추가), 상단 좌측에 **−/+ 줌·☰ 점수판·🔄 방향전환·⚙️ 설정**(작게). **이동 스틱을 끝까지 밀면 자동 달리기**. 좌/우 **동적/고정 가상 스틱**(⚙️에서 전환) — 동시 같은-반면 터치도 빈 스틱 슬롯에 배정. 스틱/버튼 시각 고도화(방향 화살표·강도 링·달리기 글로우·pressed 그라디언트). **좌우 반전(왼손잡이)** 지원.
- **모바일 설정(`client/mobile.js`, ⚙️ 패널)**: 스틱 모드(동적/고정)·사격 버튼 분리·자동사격·달리기 버튼·좌우 반전·진동·줌 감도(슬라이더). localStorage 영속, 변경 즉시 반영.
- **모바일 HUD(`client/ui.js`)**: 하단 엄지존은 컨트롤에 양보 → 체력바/탄약을 **상단 중앙**으로, 미니맵 150→104px 축소, 핑/코인/점수/순위는 좌상단. 파티클 수 감소(`particles.js`).
- **모바일/클라 안정성**: 노치/홈 인디케이터 대응(`safe-area-inset`), 데미지/사망/킬 시 **진동**, 저체력 **빨간 비네팅**, **고해상도(Retina) 캔버스**(DPR 스케일, 좌표계는 CSS px). **창 blur / 백그라운드 시 키·마우스 상태 초기화**(Alt+Tab 키 고장 방지) 및 **백그라운드 입력 송신 중지**(배터리/부정행위 방지). **네트워크 끊김 시 "재연결 중…" 오버레이**(자동 재연결). **렌더 루프는 예외 가드**(오류 시에도 다음 프레임 예약, 조용히 정지하지 않음).
- **코인 경제 + 보상형 광고**: 킬(+5)·승리(+50)로 코인 획득. 죽었을 때 **부활(광고)**(생애 1회), 로비에서 **광고 보고 SMG 시작**(라운드당 1회, 서버 `adPerkUsed` 검증). `client/ad.js`의 `showRewardedAd()`가 실제 광고 SDK로 교체될 지점(현재 스텁, 중복 호출 멱등).
- **일일 미션**: 오늘 누적 킬 `MISSION_TARGET`(8) 달성 시 보너스 코인(+100), 일간 보드와 함께 날짜(UTC) 바뀌면 초기화.
- **오늘의 랭킹(모드별 분리)**: **multi**는 킬 수 기준 일간 리더보드(코인·점수·국가 포함), **AI**는 최고 도달 레벨 기준 일간 리더보드. **보드 키는 `player.id`**(세션 단위). 각 모드 로비/결과에 맞는 패널 + 플레이 중 `Tab`(데스크탑) / `☰`(모바일) 점수판. HUD에 "내 순위 N위" 표시. 인간만 집계, AI 킬은 multi 보드·국가 순위에 섞이지 않음.
- **모드 선택 + 룸 브라우저**: 시작 화면 후 `client/mode.js` 오버레이에서 AI / 인간 멀티 선택. 멀티는 룸 브라우저로 룸별 인원수(`humans/9`·대기/플레이) 확인 후 빠른 입장 또는 특정 룸 선택 입장(`joinMulti`). 룸 목록은 `multiList` 이벤트로 갱신(시간 기반 throttle).
- **이름/국가 입력**: 접속 전 시작 오버레이(`client/start.js`)에서 닉네임 + 국가(ISO alpha-2) 선택 → `localStorage` 저장 → 소켓 `auth`로 전송 → 서버가 `sanitizeName`/`sanitizeCountry`로 검증(마크업·제어문자·비화이트리스트 국가 거부). **폼 입력 중에는 게임 키가 관여하지 않음**(스페이스/Tab/화살표 정상 동작). 모든 이름 옆에 `[KR]` 식 국가 코드 표시. **다국어(EN/KO)**: `client/i18n.js` 사전 + `t(key,vars)` + 언어 전환 옵저버(정적 DOM 갱신), 시작 화면 토글(기본 EN).
- **오늘의 국가 순위**: 일간 킬을 국가별 합산(`aggregateCountryBoard`, `shared/countries.js`) → 로비 우측 패널 "🌍 국가 순위" top 10.

## 보안 / 운영

- **정적 서빙 화이트리스트**: `/client`, `/css`, `/shared`, 루트 `index.html`만 서빙. `/server.js`, `/package.json`, `/test`, `/node_modules`, `/README.md` 등은 **404**(소스/설정/의존성 버전 노출 차단). `dotfiles: 'deny'`.
- **보안 헤더/압축**: `helmet`(CSP는 캔버스 게임 호환성 위해 비활성), `compression`, `trust proxy`(리버스 프록시 뒤용).
- **CORS**: `CORS_ORIGIN` 환경변수로 명시된 출처만 Socket.io 허용(미설정 시 동일출처만).
- **속도제한**: IP당 동시 연결 수(`MAX_CONN_PER_IP`), 전체 룸 수(`MAX_ROOMS`), 입력 최소 간격(정상 30Hz 통과, >50Hz 드랍), Socket.io `maxHttpBufferSize` 패킷 크기 제한.
- **메모리 누수 방지**: disconnect 유예 타이머 만료 시 `removeTimers`에서 핸들 해제(player 객체 클로저 GC 허용).
- **그레이스풀 셧다운**: `SIGTERM`/`SIGINT` 수신 시 `io.close()` + `httpServer.close()`로 안전 종료(PM2/컨테이너 재시작 대응). `EADDRINUSE` 등 리슨 에러 핸들링.
- **일일 리셋(UTC)**: 리더보드는 UTC 자정 기준으로 예측 가능하게 초기화.

> **참고**: 현재 구현은 "최소 방어" 단계. `clientId`는 서명/토큰 없는 난수라 위조 가능(같은 origin 전제 소규모 운영 기준), 광고 보상은 서버 횟수 제한만 있고 실제 시청 검증(SSV)은 없다. 대규모/오픈 운영 전 토큰 인증·광고 SSV 도입 필요(아래 "향후").

## 레벨 밸런스 측정 / UI 고도화

**밸런스 측정**: 헤드리스 시뮬레이터로 AI 레벨별 승률/피해/생존시간/킬을 측정해 데이터 기반 튜닝.
```bash
TRIALS=40 AIM=0.07 npm run balance   # 트라이얼 수 / 플레이어 숙련도(aimError) 조절
```
중간 실력(aimError≈0.07) 기준 현재 곡선: L1≈88% → L5≈50% → L9≈30~40% (단조적). 튜닝 지점은 `shared/player.js`의 `Bot.setSkill()`(레벨별 능력치 매핑)과 `shared/sim.js`의 `startRound()`(난이도 t 산출·시작 무기).

**UI 고도화**(`client/render.js`, `client/ui.js`, `css/style.css`):
- 엔티티: 입체 하이라이트 + 무기 금속광 + 아군(나) 흰색 글로우 테두리 + 둥근 글로스 체력바.
- 존: 펄스 글로우 테두리. 픽업: 글로우 + 부양 + 타입별 색/아이콘. 총알: 잔광(outer) + 밝은 코어(inner).
- HUD: 체력바(그라데이션·눈금·저체력 펄스), 무기 슬롯(컬러 액센트), 미니맵(프레임·존 채움·방향 화살표), 킬 피드(패널), **무기 산탄도 반영 커스텀 조준선**(데스크탑).
- 오버레이: 로비/결과 카드 패널 + **AI 레벨 진행 바(1~9)**, DOM 오버레이 버튼 호버/active 전환·배경 그라데이션.
